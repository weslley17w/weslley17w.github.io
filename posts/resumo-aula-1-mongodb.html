Resumo Aula 1 MongoDB

Nessa primeira aula nós vamos fica apenas na teoria, Resumo rapido vamos fala sobre bancos NoSQL.

Nosql são bancos não relacionais ao contrário de que muitos pensam eles não são bancos que não usa “SQL”, Acredite se quiser existe banco nosql que usa sql para fazer consultas.

Alguns exemplos de bancos nosql que usa SQL são o Casandra e o OrientDB:

vamos a alguns comandos do Cassandra

Syntax

USE mykeyspace;

CREATE TABLE users (
user_id int PRIMARY KEY,
fname text,
lname text
);


INSERT INTO users (user_id,  fname, lname)
VALUES (1745, 'john', 'smith');
INSERT INTO users (user_id,  fname, lname)
VALUES (1744, 'john', 'doe');
INSERT INTO users (user_id,  fname, lname)
VALUES (1746, 'john', 'smith');

vamos a alguns comandos do OrientDB

Syntax

SELECT [<Projections>] [FROM <Target> [LET <Assignment>*]]
[WHERE <Condition>*]
[GROUP BY <Field>*]
[ORDER BY <Fields>* [ASC|DESC] *]
[SKIP <SkipRecords>]
[LIMIT <MaxRecords>]
[FETCHPLAN <FetchPlan>]

Nota: Isso lhe lembra á alguma coisa ? “ae muleque se é pika mesmo em ! “
Claro que lhe lembra o Sql ou Mysql, postgresql, Sql Server…

E agora tiro da sua cabecinha que nosql não significa “NÃO SQL”

Então o que e nosql? ao contrário de banco relacionais que são genericos serve pra tudo que você queira criar exemplos um grafico, sistema de cache e por ai vai.

O banco nosql e diferente e feito para resolver problemas específicos por esse motivo que tem vários tipos de bancos nosql mas fica “sussa” ae que já nós chega lá, Outra grande diferença e que no banco relacional nos criamos “Regras” antes de começar à inserir dados nele, mas como assim regras Sr Weslley? Ex: Quando criamos um banco mysql para nossa aplicação, nos primeiro criamos o banco em si depois criamos tabelas e depois colunas e criamo uma regra (id -> int -> auto increment ), (name ->varchar -> length = 255) isso eu chamo de “Regra” eu acho isso bom pois não te deixa fazer merda no sistema pois não permite nos coloca um valor type text no luga de um de int exemplo, já alguns de nosso bancos nosql deixa nos fazer essa proeza o Mongo e um forte exemplo disso é também á maioria dos bancos nosql não usa joins isso eu acho chato pois temos que mudar toda nossa logica para modelar um banco nosql mas nós se acostumamos rápido. Como já falei bancos nosql são divididos em grupos:

#Chave/Valor

  • Berkeley DB
  • Tokyo Cabinet
  • Project Valdermort
  • MemcacheDB
  • SimpleDB

#Orientados a Documentos

Baseado em documentos XML ou Json, podem ser localizados pelo id único ou por qualquer registro que tenha no documento

  • CouchDB
  • MongoDB
  • Riak
  • RevenDB

#Orientado a Gráficos

São bancos orientados a gráficos ou seja serve pra cria gráfico

  • BigTable
  • Neo4J
  • Cassandra
  • HiperTable

Esses são a porra loca eles meio que são parecido com os relacional pois eles servem para para resolver varios problemas

  • OrientDB
